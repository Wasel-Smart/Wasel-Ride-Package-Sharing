# Governance And Accountability

## Accountable Roles
- Product and Mobility owner: ride conversion, corridor liquidity, feature scope
- Operations owner: packages, dispatch quality, backhaul execution
- Commercial owner: subscriptions, contracts, recurring revenue
- Trust owner: verification, disputes, support quality
- Tech owner: release quality, incidents, type safety, CI
- Design owner: shared patterns, token changes, visual consistency

## Rules
- One accountable owner per initiative
- One primary KPI per initiative
- One source of truth per workflow
- One review cadence per owner domain

## Escalation
- Red status unresolved for one cycle escalates to leadership review.
- Missing KPI or owner blocks delivery start.
- Failed verification blocks release.
