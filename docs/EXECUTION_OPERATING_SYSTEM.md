# Wasel Execution Operating System

## Objective
Move Wasel from a `6.5/10` execution maturity score to a durable `10/10` by making ownership, process, KPIs, and enforcement visible across the product and operating layers.

## Execution Standard
- No feature starts without an accountable owner, KPI, source of truth, and acceptance criteria.
- No release ships without verification status and rollback clarity.
- No incident closes without root cause and follow-up ownership.
- No strategic priority remains KPI-free.

## Accountable Domains
- Rides and corridor liquidity
- Packages and backhaul
- Wallet and monetization
- Trust and safety
- Engineering execution

## Mandatory Operating Cadence
- Weekly execution review
- Biweekly improvement cycle
- Monthly commercial review

## Required Dashboards
- Leadership scorecard: owner status, KPI drift, release risk, incident count
- Product and operations dashboard: bookings, package match rate, corridor health
- Engineering dashboard: sync failures, wallet fallback rate, type-check debt, CI status
- Commercial dashboard: Wasel Plus, commuter pass, recurring corridor revenue, contract coverage

## Enforcement
- No release with failing verification gates.
- No ownerless initiative in the delivery tracker.
- No unresolved red status older than one review cycle without escalation.
- No process change without playbook updates.
